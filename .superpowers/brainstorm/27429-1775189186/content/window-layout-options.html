独立示例窗口布局方案

目标：在编辑器内部打开一个新窗口，同时展示“原始文档”和“渲染结果”示例。

方案 A · 纯双栏
示例标题 | 类型标签 | 打开来源
原始文档
frontmatter + Markdown
渲染结果
真实预览

方案 A：纯双栏对照

最直接。打开窗口后立刻左右对照，没有额外导航。

优点

  • 心智负担最低
  • 适合只看当前一个示例

缺点

  • 切换不同示例不够顺手
  • 扩展到多类型时会开始拥挤
方案 B · 侧栏 + 双栏主体
示例窗口 | note / diary / project
示例目录
写作提示
类型切换
原始文档
原文源码
渲染结果
真实渲染

方案 B：侧栏导航 + 双栏主体

推荐。左侧负责示例与类型切换，中间和右侧稳定做原文/渲染对照。

优点

  • 适合多种文档示例
  • 窗口结构稳定，扩展性最好
  • 写作提示有自然落点

缺点

  • 比方案 A 稍复杂
  • 窗口宽度不足时要处理压缩
方案 C · 顶部标签 + 双栏 + 底部说明
note | diary | project | roadmap | graph | doc
原始文档
源码
渲染结果
预览
底部写作建议 / 字段说明

方案 C：顶部标签式

更像文档中心。强调按类型切换，但窗口内部信息层级更多。

优点

  • 类型切换明显
  • 适合和帮助页风格统一

缺点

  • 顶部和底部都占空间
  • 核心对照区域容易被挤压

我的建议

优先选 方案 B。它最适合你现在的需求：一个独立新窗口，既能稳定对照“原始文档 / 渲染结果”，又能容纳多个类别的示例文档和写作提示，不会很快长歪。
