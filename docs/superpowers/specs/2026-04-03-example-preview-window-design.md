# 示例预览独立窗口设计

## 目标

在编辑器帮助页的 `frontmatter` 分类中，为每个示例文档提供“独立新窗口预览”能力。新窗口需同时展示原始文档与真实渲染结果，帮助用户理解“写法”和“最终效果”的对应关系。

## 设计结论

- 入口保留在现有 `frontmatter` 分类，不新增新的帮助分类。
- 新窗口使用独立 Tauri 窗口，而不是系统浏览器或当前主窗口内嵌页。
- 窗口主体采用方案 A：纯双栏对照。
  - 左侧显示原始 Markdown 文档，包含 frontmatter 与正文。
  - 右侧显示真实预览结果，直接复用现有预览组件。
- 顶部仅保留极简工具栏：
  - 当前文档类型
  - 当前示例切换
  - 上一条 / 下一条
  - 复制原文
  - 同步滚动
  - 关闭窗口

## 数据来源

- 不创建第二套示例内容。
- 独立窗口直接复用 `frontmatterHelpData.ts` 中的帮助示例数据。
- 帮助页与独立窗口共享同一份示例原文，避免内容漂移。

## 技术方案

- 新窗口与主窗口共用同一前端入口，通过查询参数区分视图，例如 `?view=example-preview&exampleId=frontmatter-note`。
- 主入口在启动时判断当前是否为示例预览视图：
  - 是：渲染 `ExamplePreviewWindow`
  - 否：渲染原有编辑器应用
- 主窗口通过 Tauri `WebviewWindow` 创建或复用固定 label 的示例窗口。
- 若窗口已存在，则发事件通知切换示例并聚焦窗口，不重复创建。

## 异常处理

- 开窗失败时在主窗口明确提示错误。
- `exampleId` 无效时，独立窗口展示明确错误状态。
- 右侧预览异常时保留左侧原始文档，方便定位问题。
- 独立窗口不参与工作区、当前文件与最近打开文件状态。

## 验证

- 纯逻辑测试覆盖：
  - 路由参数解析
  - 示例选择与前后导航
- 帮助页入口测试覆盖：
  - “在新窗口中查看”入口存在
- 运行 `npm test`
- 运行 Windows 环境 `npm run build`
